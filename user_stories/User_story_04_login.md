# User story title: Place an order
Order food, Submit food order, Confirm meal order.


## Priority: 10
Must-have for iteration-1.  
This feature allows users to complete the main goal of the system by ordering food.


## Estimation: 3 days
Planning poker estimates:
* Bob: 2 days (estimated before iteration-1)
* Lisa: 3 days
* Jack: 4 days

Final agreed estimate: 3 days

## Assumptions (if any):
* The user has already selected at least one food item.
* A cart or order summary exists.
* Payment handling is out of scope for iteration-1 (order can be placed without payment).

## Description: 

Description-v1:  
As a user, I want to place an order so that I can receive food.

Description-v2:  
As a user, I want to review my selected food items and submit my order so that it can be prepared.

Description-v3:  
As a user, I want to confirm my order details before submitting so that I can avoid mistakes.


## Tasks, see chapter 4.

1. Display order summary (items, quantities, total price) – Estimation: 1 day  
2. Allow user to confirm or cancel order – Estimation: 0.5 days  
3. Submit order to backend system – Estimation: 1 day  
4. Show order confirmation message – Estimation: 0.5 days
