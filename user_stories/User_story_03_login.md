# User story title: e.g. Browse available food options
View food menu, Display available meals, Let user browse meals list.

## Priority: 10 (e.g. latest for iteration-1)
Must-have for iteration-1.  
This feature is essential so users can see what food is available before making any other actions (e.g. ordering or saving favourites).

## Estimation: 3 days
Planning poker estimates:
* Joyal: 2 days (initial estimate before iteration-1)
* Will: 3 days
* Alen: 4 days

Final agreed estimate: 3 days

## Assumptions (if any):
* The user has access to the app without needing to log in.
* Food data is already available from a database or API.
* Images for meals are optional in the first iteration.

## Description: e.g. The web page will show current deals to Orion's orbits users
Description-v1:  
As a user, I want to browse available food options so that I can choose meals easily.

Description-v2:  
As a user, I want to view a list of available meals with names and prices so that I can quickly decide what to eat.

Description-v3:  
As a user, I want to browse food options by category (e.g. breakfast, lunch, dinner) so that I can find meals faster.

## Tasks, see chapter 4.

1. Design basic food listing layout – Estimation: 0.5 days  
2. Fetch food data from database or API – Estimation: 1 day  
3. Display food name, price, and category – Estimation: 1 day  
4. Basic error handling (no food available) – Estimation: 0.5 days
