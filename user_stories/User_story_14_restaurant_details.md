# User story title: Restaurant details

## Priority: 6 (planned for iteration-2)

Viewing restaurant details is useful so users can make a better decision before ordering food.

## Estimation: 2 days

Final estimate agreed: 2 days

## Assumptions (if any)

## Refinement

- The user is logged in.
- Restaurant data (name, rating, delivery time, description) is stored in the database.
- The user has already selected a restaurant.
- Only basic restaurant information is displayed in iteration-2.

## Description
As a **user**, I want to **view restaurant details** so that **I can decide whether I want to order from that restaurant**.

### Description – version 1
The system displays a restaurant details page after the user selects a restaurant.

### Description – version 2
The restaurant details page shows the restaurant name, rating, delivery time, and description.

## Tasks (see chapter 4)
1. Design restaurant details UI layout – 0.5 days  
2. Retrieve restaurant details from database – 0.5 days  
3. Display restaurant information dynamically – 0.5 days  
4. Test restaurant details page – 0.5 days  

## UI Design
- Page title showing restaurant name
- Restaurant details section including:
  - Rating
  - Delivery time
  - Description
- Clean and structured layout
